API to HTML Documentation

API2HTML takes any data from your API and injects it in mustache templates so you can build websites without coding controllers and models.

Getting Started

The first thing you need to do is to download or clone the source code of API2HTML.

Download or clone

Installation

Once you have your code, you need to compile it from the terminal. Just cd in the directory you have cloned or downloaded and execute: 

make prepare

This command will download all the necessary dependencies, when it finishes the following command will build the binary for your platform:

make

Now you should have the binary api2html in the folder

You can move this file to /usr/local/bin or add it in any other place included in your $PATH. 

mv api2html /usr/local/bin

You can delete everything else you have downloaded as it was only needed to generate the binary.

Configure and start the server

The program only needs a configuration file to run. Create a file named config.json with all the configuration inside, you can start with this example:

{
  "robots": true,
  "sitemap": true,
  "static_txt_content": [
    "1234567890.txt"
  ],
  "pages":[
    {
      "name": "post",
      "URLPattern": "/post/:post",
      "BackendURLPattern": "https://jsonplaceholder.typicode.com/posts/:post",
      "Template": "post",
      "CacheTTL": "3600s"
    },
    {
      "name": "home",
      "URLPattern": "/",
      "BackendURLPattern": "https://jsonplaceholder.typicode.com/posts",
      "Template": "home",
      "CacheTTL": "3600s",
      "extra": {"is_home":true}
    }
  ],
  "templates": {"home":"home.mustache","post":"post.mustache"},
  "layouts": {"main":"main_layout.mustache"},
  "extra":{
    "lang": "en-US",
    "copyright": "Â© 2018 My Company"
  }
}

Now with the file created start the server

api2html serve -d -c config.json -p 8080

This will start the server in http://localhost:8080

Common problems

Error #01: json: cannot unmarshal object into Go value of type []map[string]interface {}

Usually this error happens when you are setting a response of the API as IsArray: true in the configuration file but the response is an Object. Remote the setting.

engine creation aborted: invalid character '\n' in string literal
        engine creation aborted: invalid character '"' after object key:value pair

Have you closed the quotes in the configuration file? Is a comma missing?
